# Different Kinds of Python Frameworks and Implementations

## Which Python? (Python Implementations)
how Python code runs depends on the implementation:
- **`CPython`**: The standard (what you get from python.org).
- **`PyPy`**: A faster version for CPU-heavy tasks using JIT (Just-In-Time) compilation.
- **`Jython`**/**`IronPython`**: For running Python on the Java or .NET virtual machines.

## Python Based Implementations
- **`Python libraries`**: Reusable code packages (e.g., NumPy, Django).
- **`Standalone scripts`**: Single files that perform specific tasks (e.g., `myscript.py`).
- **`Frameworks`**: Larger structures that provide tools and conventions for building applications 
  - Web applications (Django, Flask, FastAPI)
  - Desktop application (Tkinter, PyQt)
  - ...
- **`Jupyter Notebook`** / Exploratory Data Analysis
- **`Serverless function`** (AWS Lambda / GCP Cloud Function)
- **`Command-line tool (CLI)`** — argparse or click


### Python Frameworks
- **Web Frameworks**: Django, Flask, FastAPI
- **Data Science**: NumPy, Pandas, Matplotlib, Scikit-learn
- **Machine Learning**: TensorFlow, PyTorch, Keras
- **Automation**: Selenium, BeautifulSoup, Requests
- **Testing**: pytest, unittest, nose
- **GUI**: Tkinter, PyQt, Kivy
- **Game Development**: Pygame, Panda3D
- **Microservices**: Flask, FastAPI, Nameko
- **Data Visualization**: Matplotlib, Seaborn, Plotly
- **DevOps**: Ansible, Fabric, SaltStack
- **Scientific Computing**: SciPy, SymPy, Jupyter Notebooks
- **Scripting and Automation**: Python's built-in libraries (os, sys, subprocess) and third-party tools (Invoke, Click) 
- **Embedded Systems**: MicroPython, CircuitPython
